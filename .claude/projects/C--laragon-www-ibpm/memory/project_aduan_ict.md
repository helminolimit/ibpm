UC01 (Hantar Aduan ICT) for module M01 was fully implemented on 2026-04-18.

**Why:** Feature implementation based on docs/modules/mo1-aduanICT/UC01-hantar-aduan-ict.md spec.

**How to apply:** Use these patterns and table names for any follow-on UC in this module.

## Tables Created
- `kategori_aduan` — complaint categories with `unit_bpm` and `emel_unit`
- `aduan_ict` — main complaints table (FK: user_id, kategori_aduan_id), status cast to `StatusAduan` enum
- `status_log` — singular table name; model has `$table = 'status_log'`
- `lampiran_aduan` — singular table name; model has `$table = 'lampiran_aduan'`
- `notifikasi` — email notification log per aduan
- Users table extended: `bahagian`, `jawatan`, `no_telefon` columns added

## Key Files
- `app/Enums/StatusAduan.php` — Baru/DalamProses/Selesai/Ditolak/Dibatalkan with label() and color()
- `app/Enums/StatusNotifikasi.php` — Hantar/Gagal
- `app/Livewire/Permohonan/AduanIctForm.php` — 3-step form (step 1=form, 2=semakan, 3=berjaya)
- `resources/views/livewire/permohonan/aduan-ict-form.blade.php` — Flux UI multi-step form
- `app/Events/AduanDihantar.php` + `AduanBaru.php` — dispatched after successful submit
- `app/Listeners/HantarEmailPengesahan.php` + `HantarEmailNotifikasiAdmin.php` — queued mail listeners
- `app/Mail/PengesahanAduan.php` + `NotifikasiAduanBaru.php` — markdown mailables
- `resources/views/mail/pengesahan-aduan.blade.php` + `notifikasi-aduan-baru.blade.php`
- `database/seeders/KategoriAduanSeeder.php` — 7 default kategori for MOTAC
- `tests/Feature/Permohonan/AduanIctFormTest.php` — 13 Pest tests, all passing
